# Implementace i3 · Kompozitní operátor, anomálie, multi-křivkový přenos

> **Pozice v roadmapě:** iterace 3 z 5 · vyžaduje i1–i2, blokuje i4–i5
> **Rozsah:** `ToneIdentifyAndCorrect` celý pipeline (Phase A/B/C), `DeviationReport`
> analýza anomálií, multi-parameter `SplineTransfer` s mode dispatch
> **Navazuje na:** `tone_identification_and_correction.md` celý, v0.4 §4.3 (anomaly detection),
> §7.4 (SplineTransfer v0.4), virtual_test (celý dokument jako gap list)
> **Cíl:** uživatel detekuje anomálie na datasetu, opraví je jedním integrovaným operátorem
> nebo přenese křivky z jiného banku v několika parametrech najednou.

Tato iterace dodává **většinu uživatelské hodnoty celého produktu**. První dvě byly infrastrukturní;
tato přidává skutečné editační operace. Kompozice s v0.4 operátory (`PhysicalExpand`,
`ConsensusMerge`) je nativní — všechny sdílí `ApplyResult` kontrakt z i1.

## 1. User stories

### US-3.1 · Uživatel detekuje anomálie v datasetu

> *„Mám bank `Custom`, chci vidět všechny partialy, které se výrazně liší od
> konsensu ostatních dvou banků (`Partialis`, `DDSP`). Table je seřazená podle
> `|z|` sestupně, filtrovatelná per parametr, klikem na řádek se plot
> scrollne/zoomne na daný `(m, v, k)`."*

**Acceptance:**
- `GET /api/banks/:id/deviation-report?ref=bank_002,bank_003&min_z=2.0` vrátí `AnomalyReport`
- `<AnomalyList>` v left navigator: sloupce `m · v · k · parameter · z · recommend_action`
- Per-row hover highlighti odpovídající bod v `<PlotCanvas>`; click scrollne plot a otevře
  příslušnou notu
- LOO (leave-one-out) z-score per math §7.3, **ne naivní** — oprava `virtual_test #12`

### US-3.2 · Uživatel spustí Identify & Correct dry-run

> *„Vyberu notu s anomáliemi, klik `Identify & Correct`. Drawer se otevře,
> předvyplní target + reference banks. Kliknu `Preview` a vidím `reference_quality`
> score per parametr (bar chart 0–1) a distribuci plánovaných akcí (`none`,
> `soft_blend`, `hard_replace`, `fill`) jako pie chart. Ještě nic se neuložilo."*

**Acceptance:**
- `POST /api/ops/tone-identify-only` — pouze Phase A, vrací `TonalReference` payload (bez Phase B)
- `POST /api/ops/tone-identify-and-correct` s `commit=false` — celý pipeline, vrací
  diagnostics bez persistence
- `<OperatorDrawer>` zobrazí `<PreviewDiffPanel>` se stacked bar akcí + side-by-side plot
- Reference quality < 0.5 triggeruje explicit confirmation modal (řeší
  `virtual_test #19` — žádné tiché auto-downgrade)

### US-3.3 · Uživatel commitne Identify & Correct

> *„V dry-run jsem spokojen, klik `Commit`. Vznikne nový Bank, history
> crumbs ukazují nový záznam, WS invalidace propálí cache, plot se překreslí
> s opravenými hodnotami. V history detail vidím `per_partial_log` se všemi
> rozhodnutími operátoru."*

**Acceptance:**
- Druhý POST se stejnými params + `commit=true` → persist + WS emit
- `EditRecord.diagnostics.per_partial_log` obsahuje řádek pro každý `(k, parameter)`
  s akcí, hodnotou před/po, `d` (normalizovaná odchylka) a `origin`
- `<EditHistoryList>` detail má tabulku `per_partial_log` s filtrem „show only changed"

### US-3.4 · Uživatel přenese více křivek najednou (multi-parameter SplineTransfer)

> *„Chci přenést `τ₁`, `τ₂` a `A₀` z noty `(60, 5) Partialis` do not `(60, 5)`,
> `(61, 5)` a `(62, 5)` v `Custom` — ale každý parametr s jiným módem:
> `τ₁` jako `absolute` (hodnoty přímo z Partialis), `τ₂` jako `relative`
> (poměrově, zachová target fundamental), `A₀` jako `delta` (přičte korekci).
> Jeden operator call, jeden EditRecord."*

**Acceptance:**
- `POST /api/ops/spline-transfer` přijímá `parameter_configs: list[ParameterConfig]`
  (v0.4 §7.4) — každý s vlastním `mode`, `preserve_fundamental`, `clamp_to_bounds`
- Preview → diff pro všechny tři parametry v jednom panelu (tab nebo stack)
- Commit → jeden nový Bank, jeden `EditRecord` s `params=params.as_dict()`
- Backward-compat: `SplineTransferParams.single(...)` pro starý single-parameter API

### US-3.5 · Uživatel detekuje a řeší konflikt více anchorů

> *„Když pipeline narazí na dva anchors na `(k=15, τ₁)` s hodnotami,
> které se liší o víc než `3 σ₀`, chci **zablokování** operace a dialog se
> seznamem konfliktů. Možnost buď smazat jeden, nebo dočasně přepnout jeden
> na `weight=0` (disable) bez mazání."*

**Acceptance:**
- Operator raises `multiple-anchor-conflict` s payloadem konfliktních párů
- FE zobrazí `<AnchorConflictModal>` s možnostmi resolve
- Po rozhodnutí se operator replay provede automaticky

### 2.2 Phase A — pět kroků podle `tone_identification_and_correction.md §2.3`

```python
def identify_tone(
    midi: int, velocity: int,
    source_notes: list[Note],
    anchors: list[Anchor],
    *,
    parameters: list[str],
    use_physical_prior: bool = True,
    random_seed: int = 0,
) -> TonalReference:
    rng = np.random.default_rng(random_seed)
    estimates, sigmas, provenance = {}, {}, {}

    for theta in parameters:
        # Krok 1: per-source completion (bez anchors)
        per_source = [
            anchor_interpolate(
                n.partials, anchors=[], parameter=theta,
                random_seed=rng.integers(2**31),
            )
            for n in source_notes
        ]

        # Krok 2: BLUE consensus (Huber pokud |S|≥3 a outlier detekován)
        theta_cons, sigma_cons = (
            huber_consensus(per_source, k_grid=K_GRID, random_seed=rng.integers(2**31))
            if len(per_source) >= 3 and has_outlier(per_source)
            else blue_consensus(per_source, k_grid=K_GRID)
        )

        # Krok 3: Bayesian anchor update
        #   Oprava virtual_test §7.1 #2: hard anchors nepoužívají Kalman per-k,
        #   místo toho rozšíří dataset pro globální refit.
        hard_anchors = [a for a in anchors if a.parameter == theta and a.weight >= 0.999]
        soft_anchors = [a for a in anchors if a.parameter == theta and a.weight < 0.999]

        if hard_anchors:
            theta_cons, sigma_cons = refit_with_hard_anchors(
                theta_cons, sigma_cons, hard_anchors, random_seed=rng.integers(2**31),
            )
        for a in soft_anchors:
            theta_cons, sigma_cons = kalman_update(theta_cons, sigma_cons, a)

        # Krok 4: Physical regularization (pro σ > 0.2·|θ|)
        if use_physical_prior:
            theta_cons, sigma_cons = regularize_with_prior(
                theta_cons, sigma_cons, parameter=theta,
                prior_fn=physical_prior_for(theta),
            )

        # Krok 5: Clamp + monotonicity
        theta_cons = clamp_to_bounds(theta_cons, PARAM_BOUNDS[theta])
        if not is_monotone(theta_cons) and theta in POSITIVE_MONOTONE_PARAMS:
            theta_cons = enforce_pchip(theta_cons, sigma_cons)

        estimates[theta] = build_interpolator(theta_cons)
        sigmas[theta] = build_interpolator(sigma_cons)
        provenance[theta] = ProvenanceRecord(
            source_bank_ids=[n.source for n in source_notes],
            anchor_ids=[a.id for a in anchors if a.parameter == theta],
            lambda_used=…, B_hat=…, alpha_hat=…,
        )

    return TonalReference(
        midi=midi, velocity=velocity,
        f0_hz=source_notes[0].f0,   # invariant napříč zdroji (zkontrolováno ve validate)
        B=estimates["f_coef"](2) ** 2 - 1 if "f_coef" in estimates else None,
        estimates=estimates, sigmas=sigmas,
        coverage=compute_coverage(estimates, source_notes),
        quality=compute_quality(estimates, sigmas),
        provenance=provenance,
    )
```

### 2.3 Phase B — decision tree

Per-partial rozhodnutí (tone_identification_and_correction.md §3):

```python
def decide_action(
    current: float | None, ref_val: float, ref_sigma: float, ref_quality: float,
    current_sigma: float | None, is_anchored: bool,
    params: ToneCorrectionParams,
) -> Action:
    # Oprava virtual_test §7.1 #3: hard anchor reference → force hard_replace
    if is_anchored and ref_sigma < 1e-6:
        return Action(kind="hard_replace", new_value=ref_val, reason="hard_anchor")

    if current is None:
        if ref_quality < params.fill_quality_threshold:
            return Action(kind="none", reason="fill_quality_below_threshold")
        return Action(kind="fill", new_value=ref_val, reason="missing_partial")

    # Normalized deviation
    denom = math.sqrt((current_sigma or SIGMA_0[params.parameter]) ** 2 + ref_sigma ** 2)
    if denom <= 0:
        return Action(kind="none", reason="zero_denominator_guard")
    d = (current - ref_val) / denom

    if abs(d) < params.noise_threshold_d:                       # typ. 1.0
        return Action(kind="none", reason="within_noise")
    if abs(d) < params.correction_threshold_d:                  # typ. 2.5
        blend = soft_blend_weight(d, ref_quality)
        return Action(kind="soft_blend", new_value=(1-blend)*current + blend*ref_val,
                      blend_weight=blend, reason="moderate_deviation")
    return Action(kind="hard_replace", new_value=ref_val, reason="strong_anomaly")
```

**Preserve fundamental fix** (virtual_test §7.1 #4): pokud `params.preserve_fundamental=True`
a `k=1`, vrátí okamžitě `Action(kind="none", reason="preserve_fundamental")`. Default je
`True` (konzistentní se `SplineTransfer`).

### 2.4 Phase C — Bank update

Podle `tone_identification_and_correction.md §11.3`:

```python
def apply(self, bank: Bank, params: ToneCorrectionParams) -> ApplyResult:
    target = bank.get_note(params.target_note_id)
    source_notes = load_source_notes(bank, params, target.midi, target.velocity)
    anchors = bank.anchors_for_note(target.midi, target.velocity) if params.use_anchors else []

    ref = identify_tone(target.midi, target.velocity, source_notes, anchors,
                       parameters=params.parameters,
                       use_physical_prior=params.use_physical_prior,
                       random_seed=params.random_seed)

    new_note, per_partial_log = apply_correction(target, ref, params)
    new_bank = bank.with_updated_note(new_note)

    edit = EditRecord(
        operator=self.name, params=params.as_dict(),
        timestamp=datetime.now(UTC), source_note_id=None,
    )
    diag = ApplyDiagnostics(
        per_partial_log=per_partial_log,
        identification_provenance=ref.provenance,
        reference_quality=ref.quality,
        warnings=collect_warnings(ref, per_partial_log),
    )
    return ApplyResult(bank=new_bank, edit=edit, diagnostics=diag)
```

## 3. Anomaly detection — `DeviationReport`

### 3.1 Backend

```python
# piano_web/services/analysis_service.py
class AnalysisService:
    async def deviation_report(
        self, target_bank_id: str, reference_bank_ids: list[str],
        *, min_z: float = 2.0, loo: bool = True,
    ) -> AnomalyReport:
        """
        Per (m, v, k, parameter) spočítá:
          - consensus θ̂_ref(k) z reference banks (bez target, pokud loo=True)
          - σ̂_ref(k) per math §7.1
          - z = (target_value - θ̂_ref) / σ̂_ref
        Vrátí všechny záznamy s |z| >= min_z seřazené sestupně.
        """
```

**LOO z-score** (fix `virtual_test #12`): pokud `target_bank_id ∈ reference_bank_ids`,
target se z konsensu vyloučí. Pokud není, naivní z. Flag `loo` v odpovědi pro transparentnost.

## 4. `SplineTransfer` — multi-parameter mode

v0.4 §7.4 už specifikuje `parameter_configs` — v i3 se pouze implementuje a dotestuje.

### 4.1 Parameter config

```python
@dataclass(frozen=True, slots=True)
class ParameterConfig:
    parameter: Literal["tau1", "tau2", "A0", "a1", "f_coef"]
    mode: Literal["absolute", "relative", "delta"]
    preserve_fundamental: bool = True
    clamp_to_bounds: bool = True
    source_smoothing: float | None = None     # None = GCV
```

### 4.2 Mode semantics (rozšířeno z v0.4 §7.4 pseudo-code)

- `absolute` — target dostane hodnotu source splinu: `new = source_spline(k)`
- `relative` — poměrový transfer se zachováním target fundamental:
  `new = target_k1 * source_spline(k) / source_spline(1)`
- `delta` — aditivní korekce: `new = current + (source_spline(k) - source_raw(k))`

### 4.3 Backward compatibility

```python
class SplineTransferParams(BaseModel):
    # ... nové pole ...
    parameter_configs: list[ParameterConfig] | None = None

    # Legacy single-parameter API
    parameter: str | None = None
    mode: Literal["absolute", "relative", "delta"] | None = None

    @model_validator(mode="after")
    def _normalize_legacy(self):
        if self.parameter_configs is None:
            assert self.parameter and self.mode, "need either legacy OR new API"
            self.parameter_configs = [ParameterConfig(parameter=self.parameter, mode=self.mode)]
        return self

    @classmethod
    def single(cls, parameter: str, mode: str, **kwargs) -> "SplineTransferParams":
        return cls(parameter_configs=[ParameterConfig(parameter=parameter, mode=mode)], **kwargs)
```

## 9. Risks & open questions

- **R-8:** `refit_with_hard_anchors` (krok 3 fix) vyžaduje re-running celé penalized
  B-spline regrese per parametr — znásobuje cost. Mitigace: cache per-source completion
  z kroku 1 (neměníme), jen přidáme anchors do weighted LS soustavy a znovu solve
  (LAPACK call, rychlý).
- **R-9:** `Huber consensus` s `delta = 1.345·σ_MAD` není diferencovatelný → nelze
  přímo propagovat uncertainty přes `σ²_cons`. V i3 aproximujeme přes weighted BLUE
  s Huber-tuned vahami. Explicitně zdokumentované v diagnostics.
- **R-10:** `identify_tone` pro parametr `f_coef` používá uzavřený fyzikální model
  (math §3.1), ne spline — aktuální kód předpokládá spline cestu všude. Nutná
  polymorfní větev v kroku 1 nebo specialized `identify_f_coef`. V i3 řešíme přes
  strategy dispatch na parametr.
- **R-11:** Reference note `(m, v)` nemusí v jednom ze source banks existovat.
  `tone_identification_and_correction.md §11.3` to řeší `skip` s warningem; kód
  musí odlišit „bank neobsahuje notu" od „bank má notu, ale bez high-quality partials".
