§ 01

Kritická revize v0.2

Osm strukturálních problémů v původní specifikaci — od přeceněného VAE přes chybějící fyzikální model až po promíchání prezentace a dat.

01

VAE je centralizovaný, ale prakticky není potřeba pro MVP

Problém: v0.2 staví VAE jako jádro řešení (Fáze 3 z 5). Přitom trénink VAE na ~800 notách dává přetrénovaný model, KL-divergence se hůře ladí, a 95 % potřebných operací (transfer, interpolace, merge) zvládnou spliny a consensus statistika v CO prostoru bez neuronů.
Fix: VAE přesunut do Fáze 4 (volitelná). MVP používá SplineFitter + ConsensusOperator v CO prostoru. VAE je later-stage eksperiment.
02

Chybí fyzikální model piána — klíčový nástroj pro expansion

Problém: Piáno má známou fyziku — inharmonicita f_k = k·f₀·√(1+Bk²), mocninný útlum τ(k) ∝ k-α. v0.2 se tváří jako by data byla "black box" a všechno se muselo naučit z měření. Extraktory běžně detekují jen k=1..20, dalších 40 harmonik je potřeba dopočítat.
Fix: Nový modul piano_physics.py jako first-class citizen. Obsahuje: inharmonicity model, decay law fit, amplitude envelope. Použitelný jako operator PhysicalExpandOperator.
03

Chybí spline vrstva pro transfer mezi notami

Problém: v0.2 mluví o "copy decay profile", ale dělá to jako prosté kopírování per-partial hodnot. Realita: tau1_coef(k) je hladká funkce k — cubic spline fit + evaluate je správná abstrakce. Zároveň tau1_coef(midi) přes klaviaturu je také spline.
Fix: Nový modul spline_fitter.py s CubicSpline / BSpline wrappery. Transfer operace = fit na source + evaluate na target. Funguje ve 2 dimenzích: po k (harmonika) a po MIDI (klaviatura).
04

Bank je mutable — žádné undo, žádná provenience

Problém: v0.2 ukazuje bank.notes[idx] = corrected_note — přímý side-effect. Chybí edit history, nelze udělat undo, nelze sdílet intermediate stavy, nelze auditovat "kdo co změnil".
Fix: Immutable Bank. Operátor vrací (Bank_new, Edit_record). Bank má parent_id a edits: List[Edit]. Undo = vrať parent. Každá nota má provenance (od kterého extraktoru / operace pochází).
05

"VAE" je používáno současně jako zdroj dat i model

Problém: v0.2 tabulka extraktorů obsahuje "VAE → Latent Z". VAE není extraktor — je to learned model natrénovaný nad extraktory. Konceptuální match s Partialis/DDSP je mylný.
Fix: Extraktory jsou jen Partialis, DDSP, Custom. VAE je nástroj (v Operatoru VAESmoothOperator). Source string už neobsahuje "VAE".
06

Prezentace a data jsou promíchané

Problém: v0.2 frontend má "Compare Mode", "Parameter Selector" atd. jako ad-hoc UI prvky. Žádný formální model co se zobrazuje. Nelze uložit pohled, sdílet ho, ani testovat.
Fix: ViewState jako formální objekt — visible sources, parameter toggles, k-range, filter thresholds, overlay layers, comparison mode. Serializovatelný do JSON → sdílení, presety, undo/redo pohledu.
07

API je per-note, chybí batch a streaming

Problém: UC3 v0.2 dělá GET /latent/note 7× a POST /reconstruct 7× pro vyplnění velocity. Pro 1000 notů = 2000 requestů. Pro spline transfer na celé oktávě = 12 requestů.
Fix: Batch endpointy POST /api/ops/batch přijímající seznam operací. Velké operace přes WebSocket /ws/ops s progress stream. Caching CO reprezentací (Redis/in-memory) — normalizace se provádí jen při změně.
08

Anchor weight 1-9 je arbitrární ordinál

Problém: "weight 1-9" je neurčité — je to Likert škála? logaritmická? Jak ji VAE použije v loss? Navíc chybí typ anchoru — je to naměřený bod, konsenzus, nebo ruční korekce?
Fix: Anchor.weight ∈ [0, 1] (spojité) + Anchor.kind ∈ {"measured", "consensus", "manual", "physical"}. Weight se interpretuje jako inverzní variance: σ² = (1 - weight)² pro Mahalanobis loss.
Nechávám beze změny: AB ↔ CO reverzibilita, f0_hz jako invariant, per-partial struktura, MIDI/velocity rozsahy. Tyto jsou správně.

# piano_core/operators/base.py

class Operator(ABC):
    """Základní rozhraní všech editačních operací."""
    
    name: str
    
    @abstractmethod
    def apply(self, bank: Bank, **params) -> Tuple[Bank, EditRecord]:
        """Vrátí nový Bank + záznam o úpravě. Nikdy nemodifikuje vstup."""
    
    def preview(self, bank: Bank, **params) -> PreviewResult:
        """Lehká verze pro UI — bez commit, s diagnostikou."""
        new_bank, _ = self.apply(bank, **params)
        return PreviewResult(
            changed_note_ids=diff_notes(bank, new_bank),
            coef_diffs=compute_coef_diffs(bank, new_bank),
            warnings=self.validate(bank, **params).warnings
        )
    
    def validate(self, bank: Bank, **params) -> ValidationResult:
        """Pre-flight check. Vrací errors + warnings."""
        return ValidationResult(errors=[], warnings=[])
    
    def undo(self, bank: Bank) -> Bank:
        """Default: vrátí parent. Operator může overridnout pro smart undo."""
        return registry.get_bank(bank.parent_id)

§ 05

Operace A — Spline Transfer

Přenos hladkého průběhu parametru (typicky τ₁, τ₂, A₀) z jedné noty na jednu nebo více jiných. Funguje v CO prostoru přes cubic spline fit.

ALG · 5.1

Algoritmus

Vstup: Source nota A (well-extracted), target nota B (stejné nebo jiné MIDI/velocity), parametr tau1, metoda cubic_natural.

Princip: Profil tau1_coef(k) je v CO prostoru invariantní vůči MIDI a velocity (dle tvrzení spec). Fitnu spline S(k) na source's high-quality partials a vyhodnotím ho na target's k-indexech. Denormalizace se provede přes target's tau1_fund.

S(k) = CubicSpline((ki, τ₁coef[i]) | fit_qualityi ≥ θ)
τ₁target(k) = S(k) · τ₁fundtarget

Pseudo-kód

# piano_core/operators/spline_transfer.py
from scipy.interpolate import CubicSpline, UnivariateSpline

class SplineTransferOperator(Operator):
    name = "SplineTransfer"
    
    def apply(
        self, bank: Bank,
        source_note_id: str,
        target_note_ids: List[str],
        parameter: str = "tau1",           # tau1 | tau2 | A0 | a1 | beat_hz
        method: str = "cubic_natural",     # cubic_natural | cubic_clamped | bspline_smooth
        smoothing: float = 0.0,              # 0 = interpolate; >0 = smooth B-spline
        fit_quality_threshold: float = 0.7,
        k_range: Tuple[int, int] = (1, 60),
        preserve_fundamental: bool = True,   # nikdy neměnit k=1
    ) -> Tuple[Bank, EditRecord]:
        
        # 1. Source → CO
        src = bank.get_note(source_note_id)
        src_coef = src.to_coefficients()
        param_coef_field = f"{parameter}_coef"
        
        # 2. Sber body (k, coef) kde fit_quality ≥ θ
        points = [(p.k, getattr(p, param_coef_field))
                  for p in src_coef.partials
                  if p.fit_quality >= fit_quality_threshold
                  and k_range[0] <= p.k <= k_range[1]]
        
        if len(points) < 4:
            raise ValidationError("Need ≥4 high-quality partials to fit spline")
        
        # 3. Fit spline
        ks, ys = zip(*points)
        if method == "cubic_natural":
            spline = CubicSpline(ks, ys, bc_type='natural')
        elif method == "bspline_smooth":
            spline = UnivariateSpline(ks, ys, s=smoothing, k=3)
        
        # 4. Apply per target note (v CO prostoru)
        new_bank = bank
        for tid in target_note_ids:
            tgt = new_bank.get_note(tid)
            tgt_coef = tgt.to_coefficients()
            
            new_partials = []
            for p in tgt_coef.partials:
                if preserve_fundamental and p.k == 1:
                    new_partials.append(p)
                    continue
                if k_range[0] <= p.k <= k_range[1]:
                    new_val = float(spline(p.k))
                    # clamp na rozumný interval (např. [0.01, 2.0] pro τ_coef)
                    new_val = clamp(new_val, *COEF_BOUNDS[parameter])
                    p = replace(p, **{param_coef_field: new_val})
                new_partials.append(p)
            
            new_note = replace(tgt_coef, partials=tuple(new_partials)).to_note()
            # Append provenance
            new_note = append_edit(new_note, EditRecord(
                operator="SplineTransfer",
                params={"source": source_note_id, "parameter": parameter, ...},
                source_note_id=source_note_id
            ))
            new_bank = new_bank.with_updated_note(new_note)
        
        return new_bank, EditRecord(...)

§ 06

Operace B — Physical Expand

Extrapolace chybějících harmonik dle fyzikálního modelu piána. Používá inharmonicitu, mocninný útlum a amplitudovou obálku.

ALG · 6.1

Fyzikální vzorce

Motivace: Extraktory typicky detekují partials k=1..20 dobře. Pro k>20 klesá fit_quality rapidně (překrytí s noise, úzká decay). Fyzika piána je známá a umí doplnit zbytek deterministicky.

Vzorce

Inharmonicita:   fk = k · f₀ · √(1 + B · k²)
Decay law:   τ₁(k) = τ₁fund · k-α
Amplitude:   A₀(k) = A₀fund · k-β
τ₂ ratio:   τ₂(k) / τ₁(k) ≈ γ  (γ ≈ 9-12 pro piáno)

Parametry B, α, β, γ se odhadnou z dobře detekovaných partialů (log-log regrese), případně se použijí prior hodnoty z katalogu nástroje.

Algoritmus

# piano_core/operators/physical_expand.py
import numpy as np

class PhysicalExpandOperator(Operator):
    name = "PhysicalExpand"
    
    def apply(
        self, bank: Bank,
        note_id: str,
        target_k_max: int = 60,
        estimate_from_k: int = 15,         # fit na k=1..15 pak extrapoluj
        fit_quality_min: float = 0.75,
        B_override: Optional[float] = None, # inharmonicity, jinak z note.B
        gamma_tau_ratio: float = 10.0,     # τ₂/τ₁
        replace_poor_fits: bool = True,   # i přepíše existující low-quality partials
    ) -> Tuple[Bank, EditRecord]:
        
        note = bank.get_note(note_id)
        f0 = note.f0_hz
        B = B_override if B_override is not None else note.B
        
        # 1. Vyfiltruj dobré partials pro fit
        good = [p for p in note.partials
                if p.fit_quality >= fit_quality_min
                and p.k <= estimate_from_k]
        if len(good) < 5:
            raise ValidationError("Need ≥5 good partials to estimate decay laws")
        
        # 2. Log-log fit pro α (tau1 decay) a β (amplitude decay)
        ks = np.array([p.k for p in good])
        tau1s = np.array([p.tau1 for p in good])
        A0s = np.array([p.A0 for p in good])
        
        # log(τ) = log(τ_fund) - α·log(k)  →  lineární regrese
        alpha, log_tau1_fund = np.polyfit(np.log(ks), np.log(tau1s), 1)
        alpha = -alpha
        tau1_fund = np.exp(log_tau1_fund)
        
        beta, log_A0_fund = np.polyfit(np.log(ks), np.log(A0s), 1)
        beta = -beta
        A0_fund = np.exp(log_A0_fund)
        
        # 3. Generuj partials k=1..target_k_max
        existing = {p.k: p for p in note.partials}
        new_partials = []
        
        for k in range(1, target_k_max + 1):
            if k in existing and existing[k].fit_quality >= fit_quality_min:
                new_partials.append(existing[k])       # zachovej měřené
                continue
            if k in existing and not replace_poor_fits:
                new_partials.append(existing[k])       # nech i špatné
                continue
            
            # Fyzikální hodnoty
            f_k = k * f0 * np.sqrt(1 + B * k**2)
            tau1_k = tau1_fund / (k ** alpha)
            tau2_k = tau1_k * gamma_tau_ratio
            A0_k = A0_fund / (k ** beta)
            
            new_partials.append(Partial(
                k=k, f_hz=float(f_k), A0=float(A0_k),
                tau1=float(tau1_k), tau2=float(tau2_k),
                a1=0.6, beat_hz=0.0, phi=0.0,
                fit_quality=0.5     # označit jako fyzikálně odhadnuté
            ))
        
        new_note = replace(note, partials=tuple(new_partials))
        new_note = append_edit(new_note, EditRecord(
            operator="PhysicalExpand",
            params={"alpha": alpha, "beta": beta, "B": B, "gamma": gamma_tau_ratio}
        ))
        new_bank = bank.with_updated_note(new_note)
        return new_bank, EditRecord(...)

Operace C — Deviation Analysis

Porovnání N extraktů v CO prostoru. Tři úrovně metrik: per-partial variance, per-note RMSE, aggregate anomaly map.

ALG · 7.1

Algoritmus

# piano_core/analysis/deviation.py

def compute_deviation(
    banks: List[Bank],
    midi: int,
    velocity: int,
    parameters: List[str] = ["tau1_coef", "tau2_coef", "A0_coef"],
    k_range: Tuple[int, int] = (1, 60),
    z_threshold: float = 2.5,           # nad 2.5σ = anomálie
) -> DeviationReport:
    
    # 1. Normalizuj všechny banky na CO
    coefs = [b.get_note_by_midi_vel(midi, velocity).to_coefficients()
             for b in banks]
    source_names = [b.source for b in banks]
    
    per_partial = {}
    consensus = []
    anomalies = []
    
    # 2. Per-partial analysis
    for k in range(k_range[0], k_range[1] + 1):
        per_partial[k] = {}
        for param in parameters:
            values = []
            sources_present = []
            for c, name in zip(coefs, source_names):
                p = c.get_partial(k)
                if p is not None and p.fit_quality > 0.3:
                    values.append(getattr(p, param))
                    sources_present.append(name)
            if len(values) < 2:
                continue
            
            mu = np.mean(values)
            sigma = np.std(values)
            # weighted consensus = inverse-variance
            weights = np.array([1/(0.01 + p.fit_quality) for p in ...])
            consensus_val = np.average(values, weights=weights)
            
            per_partial[k][param] = {
                "mean": mu,
                "std": sigma,
                "consensus": consensus_val,
                "confidence": np.exp(-sigma / 0.05),
                "per_source": dict(zip(sources_present, values)),
            }
            
            # z-score anomaly detection
            for src, val in zip(sources_present, values):
                if sigma > 0:
                    z = abs(val - mu) / sigma
                    if z > z_threshold:
                        anomalies.append(Anomaly(
                            k=k, parameter=param, source=src,
                            value=val, consensus=mu, z_score=z
                        ))
    
    # 3. Pair-wise RMSE
    rmse_pairs = {}
    for i, j in itertools.combinations(range(len(banks)), 2):
        diffs_squared = []
        for k in range(*k_range):
            for param in parameters:
                pi = coefs[i].get_partial(k)
                pj = coefs[j].get_partial(k)
                if pi and pj:
                    d = getattr(pi, param) - getattr(pj, param)
                    diffs_squared.append(d * d)
        rmse_pairs[f"{source_names[i]}_vs_{source_names[j]}"] = np.sqrt(np.mean(diffs_squared))
    
    return DeviationReport(
        per_partial=per_partial,
        rmse_pairs=rmse_pairs,
        anomalies=anomalies,
        overall_confidence=np.mean([pp["confidence"] for ...])
    )

@dataclass(frozen=True)
class ViewState:
    """Serializovatelný pohled. Žádná data — jen konfigurace."""
    
    # === SELEKCE DAT ===
    selected_midi: int = 60
    selected_velocity: int = 5
    selected_bank_ids: Tuple[str, ...] = ()       # které banky zobrazit

    visible_parameters: FrozenSet[str] = frozenset({
        "tau1_coef", "tau2_coef", "A0_coef"
    })
    # všechny možnosti: tau1_coef, tau2_coef, A0_coef, a1,
    #                   beat_hz_coef, phi, fit_quality, f_coef
    
    # === FILTRY ===
    k_range: Tuple[int, int] = (1, 60)
    fit_quality_min: float = 0.0                  # slider 0-1
    hide_physically_estimated: bool = False      # fit_quality == 0.5 = odhad
    
    # === ŠKÁLY ===
    x_axis: str = "k"                         # "k" | "f_coef" | "log_f_coef" | "f_hz"
    y_scale: str = "linear"                    # "linear" | "log"
    y_normalized: bool = True                  # True = CO, False = AB
    
    # === OVERLAY LAYERS ===
    show_anchors: bool = True
    show_consensus_line: bool = True
    show_variance_band: bool = True             # ±σ band kolem konsenzu
    show_anomaly_markers: bool = True            # červené body nad z_threshold
    show_spline_preview: bool = False            # když je aktivní transfer
    show_physical_prediction: bool = False       # fyzikální model jako čára
    show_fit_quality_heatmap: bool = False
    
    # === COMPARISON MODE ===
    comparison_mode: str = "overlay"
    # "overlay"       — všechny zdroje překrývající v jednom grafu
    # "side_by_side"  — grid N panelů
    # "diff"          — zobraz odchylky od reference
    # "consensus"     — jen konsenzus + variance band
    diff_reference_bank: Optional[str] = None
    
    # === EDIT MODE ===
    edit_mode: str = "view"
    # "view"           — jen čtení
    # "anchor_place"   — klik přidává anchor
    # "spline_edit"    — drag bodů spline křivky

    # === COLORING ===
    color_by: str = "source"    # "source" | "fit_quality" | "confidence" | "provenance"
    
    def toggle_parameter(self, param: str) -> "ViewState":
        new_set = self.visible_parameters ^ {param}
        return replace(self, visible_parameters=new_set)
    
    def to_dict(self) -> dict: ...
    def from_dict(d: dict) -> "ViewState": ...

Závěr

Architektura, která je ready pro kód, s jasnými vrstvami a operátory jako zvukový "Unix philosophy".

v0.3 je kompaktnější než v0.2 navzdory tomu, že přidává tři hlavní operace a ViewState vrstvu. Důvod: redukce VAE z centra na volitelnou komponentu uvolnila architektonický prostor, který byl nahrazen praktičtějšími abstrakcemi (Operator pattern, immutable Bank, piano_physics).

Doporučený start: Fáze 1 + 2 paralelně. F1 dodá core model a I/O, F2 začne implementací SplineTransferOperator jako prvního operátoru — ten je nejsnazší validovat proti existujícím datům a zároveň je to nejhodnotnější operace pro editor.

Kritérium hotovosti MVP: Načíst 3 banky (Partialis, DDSP, Custom) → spustit deviation analýzu → identifikovat anomálie → aplikovat spline transfer na opravu → exportovat konsolidovanou banku. Celý flow bez VAE, bez Electronu, čistě v prohlížeči.
