# Implementace i5 · 3D vizualizace, integrita banku, export

> **Pozice v roadmapě:** iterace 5 z 5 · vyžaduje i1–i4, blokuje pouze v1.0 release
> **Rozsah:** keyboard-wide 3D surface plot, `BankIntegrityOperator` (inter-note validator),
> export do tří formátů (ICR, synth-friendly, analýza CSV), deterministic replay test-suite
> **Navazuje na:** v0.4 §7.5 (katalog operátorů), virtual_test §7.2 #12 (inter-note validator),
> §7.3 #11 (hybridní operator chains), §8 (reprodukovatelnost)
> **Cíl:** uživatel vizualizuje celou klaviaturu najednou, ověří integritu úprav napříč
> všemi notami a vyexportuje bank v produkčním formátu s kompletní audit trail.

Tato iterace uzavírá editační smyčku: od bodového pohledu (jedna nota, jeden parametr
v i1) až po celkový pohled (88 not × 60 partialů v 3D surface). Zároveň finalizuje
reprodukovatelnost — každá úprava v celém produktu musí být přehrajitelná bit-for-bit.

## 1. User stories

### US-5.1 · Uživatel vidí 3D surface plot klaviatury

> *„Chci pro zvolený parametr (např. `τ₁`) a velocity (např. 5) vidět
> 3D surface: x = midi (21..108), y = k (1..60), z = hodnota parametru.
> Můžu otáčet kamerou, zoomovat, klikat na body a dostat detail. Extrapolované
> partialy jsou barevně odlišené (šedé místo barevné škály). Anchors jsou 3D
> markery nad surface."*

**Acceptance:**
- `<Surface3DPlot>` komponenta (Plotly surface) v novém tabu `3D`
- Volby: `parameter` (radio), `velocity` (slider), `color_by` (value | fit_quality | origin)
- Režim `difference`: zobrazí `current_bank - parent_bank` pro vizualizaci efektu edits
- Performance: < 500 ms render pro 88×60 mřížku (první render)

### US-5.2 · Uživatel identifikuje keyboard-wide problémy

> *„Spustím `BankIntegrity.validate` na celý bank. Dostanu report:
> non-monotonic sekvence, chybějící noty, partialy s quality < threshold,
> fyzikálně nekonzistentní hodnoty (např. `τ₁ > τ₂` na některých k).
> Klik na issue → 3D plot highlighti přesné místo, optional 'Fix it' button
> spustí doporučený operátor."*

**Acceptance:**
- `POST /api/ops/bank-integrity-validate` (preview-only, nic nepíše)
- `<IntegrityIssuesPanel>` v left navigator (vedle AnomalyList)
- Fix-it button pro typické issues: `ToneIdentifyAndCorrect` pro anomaly,
  `PhysicalExpand` pro missing tail, `AnchorInterpolate` pro non-monotonic

### US-5.3 · Uživatel exportuje bank do ICR

> *„Klik `Export → ICR JSON`. Dostanu soubor `bank_XXX_vN.icr.json` s kompletními
> partialy, provenance, edit history, anchors. Stejný soubor lze nahrát zpět
> (round-trip) a bank bude bit-identický (modulo timestamps)."*

**Acceptance:**
- `GET /api/banks/:id/export?format=icr` streamuje JSONL (scalable)
- Ověřeno testem: `import(export(bank)) == bank` na metric-level hash
- Schema version pole, backward compat s ICR v1

### US-5.4 · Uživatel exportuje do synth-friendly formátu

> *„Pro načtení do Pianoteq nebo vlastního synthu chci flattened formát:
> per nota, per partial, absolutní hodnoty (ne coefficienty). Volby:
> CSV per nota, nebo jeden velký CSV. Možnost vyloučit extrapolované partialy."*

**Acceptance:**
- `GET /api/banks/:id/export?format=synth_csv&exclude_extrapolated=true`
- Sloupce: `midi, velocity, k, f_hz, A0, tau1, tau2, a1, beat_hz, phi, origin`
- Coefficients → absolute hodnoty přes `NoteCoef.to_note()` / inverse

### US-5.5 · Uživatel spustí deterministic replay test

> *„V dev módu se mi objeví tlačítko `Verify reproducibility`. Systém projde
> celou edit history, replay ji s persistovanými parameters + seedy a porovná
> s aktuálním state. Report ukáže, jestli je bank bit-identický, nebo kde
> jsou odchylky."*

**Acceptance:**
- `POST /api/banks/:id/verify-reproducibility` (admin-only endpoint)
- Report per edit: `(edit_id, match, hash_expected, hash_actual, max_delta)`
- Pokud mismatch, indikuje pravděpodobnou příčinu (non-deterministic step,
  dependency version change)

## 2. 3D vizualizace

### 2.1 `<Surface3DPlot>` technický design

Plotly `Surface` trace s `x, y, z` = `midi, k, value`. Color dimension podle
user volby. React komponenta drží kameru state přes Zustand (persist přes
relogin).

```tsx
interface Surface3DProps {
  bankId: string;
  parameter: ParameterName;
  velocity: number;
  colorBy: "value" | "fit_quality" | "origin";
  differenceFrom?: string;       // parent bank ID pro diff mode
}
```

### 2.2 Data preparation

```python
# piano_web/services/visualization_service.py
class VisualizationService:
    async def surface_grid(
        self, bank_id: str, parameter: str, velocity: int,
        *, color_by: str, difference_from: str | None = None,
    ) -> SurfaceGrid:
        bank = await self._load(bank_id)
        midis = sorted({n.midi for n in bank.notes if n.velocity == velocity})
        ks = list(range(1, K_MAX + 1))

        z = np.full((len(midis), len(ks)), np.nan)
        color = np.full((len(midis), len(ks)), np.nan)
        origin_mask = np.zeros((len(midis), len(ks)), dtype="U16")

        for i, m in enumerate(midis):
            note = bank.get_note_by_midi_vel(m, velocity)
            for p in note.partials:
                if p.k <= K_MAX:
                    z[i, p.k - 1] = getattr(p, parameter)
                    origin_mask[i, p.k - 1] = p.origin
                    color[i, p.k - 1] = self._color_value(p, color_by)

        if difference_from:
            parent = await self._load(difference_from)
            z_parent = self._build_grid(parent, parameter, velocity, midis, ks)
            z = z - z_parent

        return SurfaceGrid(
            midi=midis, k=ks, z=z.tolist(), color=color.tolist(),
            origin_mask=origin_mask.tolist(),
        )
```

### 2.3 Performance

- 88 × 60 = 5280 cells → Plotly surface zvládá nativně < 200 ms
- Re-render při změně `velocity` pouze swap z-matrix (zachovat kameru)
- Při diff mode cachovat grid parent-bank (immutable) per bank_id

### 2.4 Interakce

- `Hover bod` → tooltip `midi · k · value · origin · fit_quality`
- `Double-click bod` → open note + open parameter v 2D plot
- `Right-click` → menu `Set as anchor`, `Show in AnomalyList`, `Add to selection`
- `Camera presets`: top-down (like heatmap), side (k-vs-value), ISO

## 3. `BankIntegrityOperator`

### 3.1 Specifikace (virtual_test §7.2 #12)

Validator, který neprovádí úpravy — jen reportuje. Implementován jako `Operator`
subclass s `apply()` vracejícím nepozměněný bank, ale naplněným `diagnostics`.

```python
# piano_core/operators/bank_integrity/operator.py
class BankIntegrityOperator(Operator):
    name = "BankIntegrity"
    params_class = BankIntegrityParams

    def validate(self, bank: Bank, params: BankIntegrityParams) -> ValidationResult:
        return ValidationResult(ok=True, errors=[], warnings=[])

    def apply(self, bank: Bank, params: BankIntegrityParams) -> ApplyResult:
        issues = []
        issues.extend(self._check_monotonicity(bank, params))
        issues.extend(self._check_missing_notes(bank, params))
        issues.extend(self._check_quality_floor(bank, params))
        issues.extend(self._check_physical_consistency(bank, params))
        issues.extend(self._check_tau_ordering(bank))

        diag = ApplyDiagnostics(integrity_issues=issues, ok=len(issues) == 0)
        edit = EditRecord(
            operator=self.name, params=params.as_dict(),
            timestamp=datetime.now(UTC), source_note_id=None,
        )
        return ApplyResult(bank=bank, edit=edit, diagnostics=diag)
```

### 3.2 Checks

| Check | Popis | Severity |
|---|---|---|
| Monotonicity | τ₁, τ₂, A₀ napříč MIDI pro fixní (v, k) | warn / error podle délky sekvence |
| Missing notes | Chybějící `(midi, velocity)` v rámci očekávaného gridu | warn |
| Quality floor | `fit_quality < 0.3` pro více než N partialů v notě | warn |
| Physical consistency | `τ₂ < τ₁` (pro piano dekády je τ₂ > τ₁) | error |
| Inharmonicity | `B < 0` nebo `B > 0.01` (mimo physical range) | error |
| τ ordering | τ₁ monotónní klesání v rámci noty jako funkce k | warn |

### 3.3 `IntegrityIssue` schema

```python
class IntegrityIssue(BaseModel):
    kind: Literal["monotonicity", "missing_note", "quality_floor",
                  "physical_inconsistency", "inharmonicity_range", "tau_ordering"]
    severity: Literal["error", "warning", "info"]
    location: IssueLocation            # (midi, velocity, k?, parameter?)
    detail: str
    suggested_operator: str | None     # "ToneIdentifyAndCorrect", etc.
    suggested_params: dict | None
```

def test_icr_roundtrip_preserves_everything():

## 5. Deterministic replay test suite

### 5.1 Co testujeme

Pro každý operátor a každý typ edge case:

1. **Idempotency**: `op.apply(bank, p) == op.apply(bank, p)` při stejném seedu
2. **Replay**: `replay(edit_history) == current_state`
3. **Version stability**: import banku exportovaného z i3 musí fungovat v i5+
4. **Seed sensitivity**: různý seed → různé (ale valid) výsledky; stejný seed → identické

### 5.2 Implementace replay endpointu

```python
async def verify_reproducibility(bank_id: str) -> ReplayReport:
    current_bank = await bank_repo.load(bank_id)
    root_bank = await _find_root(current_bank)

    replayed_bank = root_bank
    report_items = []
    for edit in current_bank.edit_history:
        op = OperatorRegistry.get(edit.operator)
        params = op.params_class()(**edit.params)
        result = op.apply(replayed_bank, params)

        expected_hash = _hash_bank_at_edit(current_bank, edit.edit_id)
        actual_hash = bank_hash(result.bank)

        report_items.append(ReplayItem(
            edit_id=edit.edit_id,
            operator=edit.operator,
            match=expected_hash == actual_hash,
            hash_expected=expected_hash,
            hash_actual=actual_hash,
            max_delta=None if expected_hash == actual_hash else _compute_max_delta(...),
        ))
        replayed_bank = result.bank

    return ReplayReport(
        bank_id=bank_id,
        items=report_items,
        final_match=bank_hash(replayed_bank) == bank_hash(current_bank),
    )
```

### 5.3 Známé non-deterministic body a jejich řešení

| Krok | Non-det zdroj | Fix |
|---|---|---|
| GCV grid search | `np.argmin` u tie values | Deterministic tie-break: smallest λ |
| Bootstrap uncertainty | `np.random` | `rng = np.random.default_rng(seed)` |
| Huber delta tuning | iterative MAD convergence | Fixed max iterations + seed-driven init |
| BLAS/LAPACK rounding | threading | Single-thread pro scipy solves v edit path |
| Dict ordering | Python 3.12 je insertion-order, ale hash seeds | `PYTHONHASHSEED=0` v edit replay |

## 9. Risks & open questions

- **R-17:** Plotly 3D surface s velkými daty může v prohlížeči zatěžovat GPU.
  Pro 88×60 grid OK, ale pokud se v budoucnu zvýší `K_MAX` nad 60, bude potřeba
  decimation nebo přechod na WebGL (deck.gl). V i5 držíme Plotly.
- **R-18:** `BankIntegrityOperator.apply` vrací původní bank, ale `EditRecord`
  se normálně ukládá — každý validate tedy generuje edit entry. Alternativa:
  zavést `ValidationOperator` mimo hierarchii `Operator`, který nemění bank
  a negeneruje edit. Ale to by znamenalo druhou kategorii, zvyšuje complexity.
  V i5 ponecháváme jako Operator s `no_op=true` flagem v EditRecord;
  UI takové edits skrývá z history.
- **R-19:** Deterministic replay předpokládá, že všechny verze závislostí
  jsou pinned. Upgrade scipy/numpy může mikro-změnit výstup. Mitigace:
  lock file v repo + CI check, že replay na fixture datech projde při
  každé dep upgrade PR.
- **R-20:** Export s `include_history=true` může být velký (MBy pro dlouhé
  edit chains). Pro small user export asi OK, pro production archive
  preferujeme `icr_ndjson` streaming. V i5 obě podporujeme.
- **R-21:** Pianoteq synth CSV formát není oficiálně specifikovaný — máme
  reverse-engineered layout z dokumentace a příkladů. Pokud se formát mění
  mezi Pianoteq verzemi, náš export se rozbije. Mitigace: version selector
  v ExportWizard (`Pianoteq 7`, `Pianoteq 8`), default na nejnovější.

## 10. Post-i5 — co zbývá pro v1.0 release

Mimo scope pěti iterací, ale potřebné před veřejným v1.0:

- **Authorization & multi-tenancy**: v0.4 záměrně neřeší, iterace jsou single-tenant.
  Integrace přes OAuth2 + per-bank RBAC až po i5.
- **Correlation matrix Σ** napříč zdroji (virtual_test §7.2 #8) — i3 ponechává
  diagonální BLUE. Pro v1.0 implementovat `estimate_cross_source_correlation`
  a nediagonální fúzi.
- **Hybridní operator chains** (virtual_test §7.3 #11) — template `ToneCorrectThenTransfer`
  pro časté use case. Nad `execute_batch` z v0.4 §8.1, ale s preset chain definicemi.
- **Undo/redo UI** přes git-like branching — core immutable chain už to umožňuje,
  UI zatím nemá explicit branch viewer.
- **User preferences persistence**: threshold defaults, parameter visibility,
  camera presets across sessions.
